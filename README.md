# חדר בריחה

חדר בריחה אונליין קצר: כל שלב נפתח לפי הסדר, המשתתפים מקלידים קוד מספרי, תשובה נכונה מגלה חלק מהסיסמה הסופית, ובסוף מרכיבים את כל החלקים כדי לפתוח את דף הניצחון.

האתר בנוי ב-React + Vite עם Netlify Functions קטנות לבדיקת תשובות בצד שרת ואזור ניהול לעריכת המשחק.

## מה יש באתר

- שלבי משחק שניתנים לעריכה, הוספה ומחיקה דרך Admin
- דף סופי: `/final`
- ממשק בעברית ו-RTL מלא
- תמיכה בשאלות מודפסות ליד ה-QR או שאלות שמופיעות באתר
- קלט מספרי לכל שלב
- אנימציית הצלחה עם חשיפת אות או חלק מהקוד
- אנימציית טעות וניסיון חוזר
- פתיחת שלבים לפי סדר עם מנעול מונפש
- בדיקת תשובות וקוד סופי בצד שרת
- אזור Admin לעריכת שאלות, תשובות, הודעות, חלקי קוד, שלבים והפתרון הסופי
- שמירת התקדמות בלבד בדפדפן
- כפתור מצב כהה
- קונפיגורציה מוכנה ל-Netlify Functions + Netlify Blobs

## איך המשחק עובד

1. מדפיסים לכל שלב QR שמוביל לכתובת ייחודית.
2. ליד כל QR אפשר להדפיס את השאלה או הרמז.
3. המשתתף נכנס לעמוד, מקליד את התשובה המספרית, ומקבל חלק מהקוד הסופי.
4. אחרי כל השלבים, נכנסים ל-`/final` ומקלידים את הקוד שנאסף.
5. אם הקוד נכון, מופיע מסך ניצחון.

## כתובות ל-QR

אחרי העלאה ל-Netlify, כל QR יכול לפתוח אחת מהכתובות של השלבים:

```text
/q/1
/q/2
/q/3
/q/4
/q/5
/final
```

אם מוסיפים שלב דרך Admin, הוא יקבל כתובת חדשה כמו `/q/6`.

לדוגמה, אם האתר עלה לכתובת:

```text
https://your-site-name.netlify.app
```

אז ה-QR הראשון צריך להוביל אל:

```text
https://your-site-name.netlify.app/q/1
```

## הרצה מקומית

צריך Node.js ו-npm.

```bash
npm install
npm run dev
```

אחרי ההרצה, האתר ייפתח מקומית בכתובת שמופיעה בטרמינל, בדרך כלל:

```text
http://127.0.0.1:5173/
```

`npm run dev` מריץ את ה-frontend בלבד. כדי לבדוק גם Functions מקומית אפשר להשתמש ב-Netlify CLI:

```bash
netlify dev
```

## בדיקת Build

לפני העלאה ל-Netlify מומלץ להריץ:

```bash
npm run build
```

הפקודה יוצרת את תיקיית `dist`, שהיא תיקיית הפריסה של האתר.

## עריכת שאלות, תשובות וקודים

עריכת המשחק נעשית דרך:

```text
/admin
```

באזור הניהול אפשר לשנות:

- שם המשחק וטקסט הפתיחה
- הטקסט לפני הקוד הסופי
- הפתרון הסופי
- הודעת הצלחה ושגיאה ברירת מחדל
- הודעת שגיאה בקוד הסופי
- טקסטים של מסך הסיום
- הוספה ומחיקה של שלבים
- `title` - שם השלב שמופיע באתר.
- `question` - השאלה שתופיע באתר. אם משאירים מחרוזת ריקה, האתר מציג הודעה שהשאלה נמצאת ליד ה-QR המודפס.
- `answer` - המספר הנכון שהמשתתף צריך להקליד.
- `reward` - האות או חלק הקוד שנחשף אחרי תשובה נכונה.
- `successMessage` - הודעת הצלחה ספציפית לשלב. אם ריק, משתמשים בהודעת ברירת המחדל.
- `errorMessage` - הודעת שגיאה ספציפית לשלב. אם ריק, משתמשים בהודעת ברירת המחדל.

השינויים נשמרים ב-Netlify Blobs, כך שאין צורך לערוך קוד או לפרוס מחדש אחרי שינוי תוכן. אחרי שמירה, ההתקדמות בדפדפן הנוכחי מתאפסת כדי שאפשר יהיה לבדוק מיד את התשובות החדשות.

הקובץ `src/data/challenges.js` מכיל רק fallback ציבורי לתצוגה. הוא לא מכיל תשובות או פתרון סופי.

אם רוצים שהשאלה תהיה רק בדף המודפס, משאירים:

```text
שאלה באתר: ריק
```

## הגדרות Admin

כדי שאזור הניהול יעבוד ב-Netlify, צריך להגדיר שני משתני סביבה:

```text
ADMIN_PASSWORD
ADMIN_TOKEN_SECRET
```

אפשר להגדיר אותם דרך Netlify Dashboard או CLI:

```bash
netlify env:set ADMIN_PASSWORD "your-password"
netlify env:set ADMIN_TOKEN_SECRET "long-random-secret"
```

אחרי שינוי משתני סביבה צריך לבצע redeploy.

## פריסה ב-Netlify

הפרויקט כבר כולל `netlify.toml` וקובץ redirects, כך שכתובות כמו `/q/1` ו-`/final` יעבדו גם אחרי רענון הדף.

הגדרות מומלצות ב-Netlify:

```text
Build command: npm run build
Publish directory: dist
Functions directory: netlify/functions
```

אם מחברים את Netlify ל-GitHub, כל push ל-repo יכול להפעיל פריסה חדשה אוטומטית.

פריסה ידנית עם Netlify CLI:

```bash
npm run build
netlify deploy --prod --dir dist --functions netlify/functions
```

## אבטחה

התשובות והפתרון הסופי לא נשלחים ל-frontend ולא נשמרים ב-localStorage. הדפדפן שומר רק התקדמות, למשל אילו שלבים כבר נפתרו.

הבדיקה נעשית דרך Netlify Functions:

- `public-config` מחזיר רק מידע ציבורי.
- `check-answer` בודק תשובת שלב.
- `check-final` בודק את הפתרון הסופי.
- `admin-login` ו-`admin-config` מנהלים את אזור הניהול.

זה מתאים למשחק קליל או אירוע משפחתי. אם צריך אבטחה חזקה יותר, אפשר להוסיף בעתיד משתמשים אמיתיים, הגבלת ניסיונות, קודים חד-פעמיים או audit log.
